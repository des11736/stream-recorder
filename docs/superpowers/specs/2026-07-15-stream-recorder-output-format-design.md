# 流收录程序：异常检测与输出格式简化设计

## 目标

简化新建收录任务界面，移除用户可配置的 RTSP 传输与画面分析选项；允许用户在 HLS（m3u8）与连续 MP4 两种收录格式间选择，并保留可选的通用异常检测。

## 界面

- 移除“RTSP 传输”下拉框；RTSP 输入在内部固定使用 TCP。
- 移除“画面分析”选项，不提供黑屏或画面冻结检测。
- 增加一个可切换按钮“异常检测：开启/关闭”，默认开启。
  - 开启时记录丢帧/丢包、断流、无有效画面和收录进程异常。
  - 关闭时保留任务状态和自动重连，隐藏上述详细异常提示。
- 增加“收录格式”下拉框：`m3u8（HLS）` 与 `MP4`。
- 将“分片时长”改为“切片长度”。仅选择 m3u8 时显示；选择 MP4 时隐藏该标签和输入框。
- 将路径选择按钮缩小为“选择”。

## 收录行为

### m3u8（默认）

- 输出 `index.m3u8`、`init.mp4` 与 fMP4 分片。
- 使用“切片长度”作为 HLS 分片时长，范围仍为 2 至 60 秒。

### MP4

- 在每项任务独立目录中持续写入 `recording.mp4`。
- 使用 fragmented MP4 标志，使录制期间文件可被播放器读取。
- 视频始终使用流拷贝；AAC 音频流拷贝，其他音频编码为 AAC。

## 异常检测

- 不再创建黑屏/冻结滤镜或输出相应日志。
- 异常检测开启时，额外使用低帧率视频监视器识别“无有效画面”；FFmpeg 主收录输出仍用于识别丢帧、丢包与断流。
- 异常检测关闭时，监视器不启动；收录任务仍会自动重连，且状态改变继续显示。

## 验证

- 单元测试：格式选择、MP4 命令、异常检测开关、无黑屏/冻结事件、界面控件显隐与禁用状态。
- 集成测试：H.264 与 H.265 分别生成 HLS 和 MP4 输出。
- 打包验证：正式 EXE 可正常打开主界面。
