# 运行任务表格对齐与停止按钮设计

## 目标

修正“运行任务”表格中右侧“操作”列过度拉伸造成的视觉断层；停止按钮采用固定紧凑尺寸，并移除操作单元格中与行底色不一致的背景。

## 根因

任务 ID 列被隐藏后，表头的 `setStretchLastSection(True)` 会把最后一个可见列“操作”扩展为剩余宽度。停止按钮被直接作为该单元格组件设置，因此也随操作列横向填满；组件默认绘制的底色与交替行背景不一致。

## 设计

### 表格列策略

- 取消“最后一列自动拉伸”。
- 将“操作”列设置为固定 120px；表头和每一行的操作单元格均使用同一列宽。
- 将“最新事件”列设置为唯一的伸展列，用于吸收窗口宽度变化。
- 保留已有其他列的默认宽度、输出目录宽度和隐藏任务 ID，不改变任务数据或排序/选择行为。

### 停止操作单元格

- 在“操作”单元格中放入透明容器，其布局零边距并居中对齐。
- 停止按钮固定为 96px × 30px，继续使用既有次要按钮样式和停止信号连接。
- 容器不填充背景，交替行底色由表格本身连续绘制，不产生按钮下方或两侧的额外色块。

## 边界与非目标

- 不修改录制、停止、状态更新、日志、FFmpeg 或任务数据逻辑。
- 不调整新建任务表单与其他按钮样式。
- 保留任务 ID 隐藏列。

## 验收与验证

- UI 测试验证：操作列为固定 120px、最新事件列为伸展列、最后一列不再自动伸展。
- UI 测试验证：停止按钮位于透明居中容器中，最小/最大尺寸均为 96px × 30px，点击仍发出正确的任务 ID。
- 运行完整 pytest 测试集。
- 生成并启动新的 Windows EXE，确认窗口可正常打开。
