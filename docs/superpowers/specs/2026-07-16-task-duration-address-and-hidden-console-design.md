# 任务时长、地址记忆与隐藏子进程窗口设计

## 目标

让任务列表中的“已收录”时长在实际收录期间递增、停止后保留；重启程序后自动填回上一次成功启动的流地址；在 Windows 上启动 FFprobe 和 FFmpeg 时不再弹出空白 CMD 窗口。

## 范围

- 地址仅在通过校验并点击“启动收录”后保存，程序下次启动自动回填完整地址。
- 使用当前 Windows 用户的 `QSettings` 保存地址，不创建额外配置文件。
- 每个任务独立累计时长：收到“收录中”状态时开始或恢复计时；“正在检测”和“正在重连”不计时；“已停止”或“失败”时冻结最终值。
- 时长以 `HH:MM:SS` 显示，并由主界面每秒刷新；多个任务互不影响。
- Windows 上对 FFprobe、主收录 FFmpeg 和异常检测 FFmpeg 统一使用 `CREATE_NO_WINDOW`；非 Windows 平台不传该参数。
- 原有日志采集、进程终止、地址脱敏和多任务行为保持不变。

## 结构与数据流

1. 主窗口创建时从 `QSettings` 读取最后保存的地址并填入地址框。
2. 用户成功发起任务时，主窗口保存规范化后的地址，并创建该任务的独立计时状态。
3. 服务层把收录器状态写入事件队列；主窗口消费状态事件后驱动该任务的计时状态。
4. 主窗口每秒刷新处于“收录中”任务的时长单元格。重连期间暂停，结束状态到来后不再变化。
5. 进程启动层根据平台生成子进程参数；Windows 返回隐藏控制台参数，FFprobe 与两类 FFmpeg 进程均复用该参数。

## 测试

- 使用临时 INI 格式 `QSettings` 验证地址在新窗口创建时恢复，且无效地址不覆盖已有值。
- 验证任务在“收录中”累计、在重连时暂停、停止后冻结并保留显示值。
- 验证 Windows 平台返回 `CREATE_NO_WINDOW`，其他平台返回空参数；验证探测与两种收录进程均使用该参数。
- 执行现有完整测试集和打包后程序启动检查。

## 不包含

- 不保存任务列表或正在运行的任务；关闭程序仍会停止它们。
- 不把地址写入日志，也不修改现有地址脱敏规则。
- 不根据 FFmpeg 的媒体时间计时，以避免自动重连后时长回退或重置。
