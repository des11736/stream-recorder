# 流收录程序

Windows 桌面程序：填写 RTMP、RTSP、HTTP 或 HTTPS 流地址，点击“启动收录”即可将 H.264/H.265 视频封装为 HLS。

## 功能

- 一次可运行多个独立收录任务。
- 程序会在下次启动时，自动填回上一次成功启动收录的流地址。
- 启动前自动检测视频编码，只接受 H.264 与 H.265。
- 可选择输出 m3u8（fMP4 HLS）或持续写入的 MP4 文件。
- m3u8 任务输出独立的 index.m3u8、init.mp4 和 fMP4 分片；MP4 任务输出 recording.mp4。
- 不重编码视频，保留源视频质量并降低 CPU 占用。
- 任务列表“已收录”仅在“收录中”状态累计；检测和自动重连期间不累计，停止或失败后会保留最终时长。
- 收录过程中实时显示中文日志，并在任务输出目录的 logs 文件夹落盘保存。
- 异常检测开启时，记录接收端丢帧/丢包、断流、自动重连、无有效画面、编码异常和写入异常。
- Windows 上的 FFprobe、主收录 FFmpeg 与异常检测 FFmpeg 均不会显示 CMD 窗口；日志仍可在界面和输出目录日志文件中查看。
- 地址中的密码、token、签名参数会在可见日志中脱敏。

## 运行

开发环境：

    python -m pip install -e ".[dev]"
    python -m stream_recorder.main

打包单文件 EXE：

    .\scripts\build.ps1

构建脚本会下载并校验 FFmpeg 8.1.2 Full Build，然后生成 dist\流收录程序.exe。最终 EXE 在启动时自动释放内置 FFmpeg/FFprobe，用户不需要额外安装。

## 使用

1. 填写任务名称和流地址；常见误写的 rmtp:// 会自动改为 rtmp://。成功启动收录后，流地址会保存，并在下次启动程序时自动回填。
2. 选择输出位置，选择 m3u8 或 MP4 收录格式。
3. 选择 m3u8 时，可设置 2 到 60 秒的“切片长度”；MP4 为连续单文件，会隐藏该选项。
4. 默认开启“异常检测”，需要时可关闭；RTSP 内部固定使用 TCP。
5. 点击“启动收录”。编码检测成功后，任务开始生成 index.m3u8 或 recording.mp4；任务列表“已收录”只在“收录中”状态递增，检测或重连时暂停计时，停止或失败后保留最终时长。
6. 在“实时中文日志”区域检查状态，也可在任务输出目录的 logs 文件夹查看完整日志。Windows 不会为 FFprobe、主收录 FFmpeg 或异常检测 FFmpeg 弹出 CMD 窗口。

异常检测开启时会额外建立一个低帧率监视连接，用于发现无有效画面；不检测黑屏或冻结。源端并发连接有限时可以关闭“异常检测”。

## 许可证与授权

本程序仅用于收录你有权访问的流，不绕过任何 DRM、鉴权或访问控制。随包附带 FFmpeg 的许可证；FFmpeg Windows 构建来源为 GyanD，版本固定为 8.1.2，下载 SHA-256 校验值由 Windows Package Manager 清单提供。
